# 档案数据批管开发模板

> **定位**：与花名册员工档案 **同表同规则**，入口为「组织与员工 › 管理数据」跨员工批管（列表 CRUD + 导入/导出）。  
> **参考实现**：证件信息（`id-documents`）。接入其余 25 个 resource 时 **禁止复制 26 套业务校验**，须遵循本文清单。  
> **契约**：`shared/api.interface.ts` → `ArchiveDataApi`；路由 `/admin/employees/data/:resource`。

---

## 1. 架构一览

```text
菜单 / 权限（已就绪，勿重复建表）
  └─ V59 group_archive_data + 5 子 GROUP + 26 ITEM

前端
  ├─ config/archive-data-resources.ts     # supported + 列表列 + 表单字段 + importConfig
  ├─ pages/.../AdminArchiveDataPage.tsx   # 泛型壳（列表 / Sheet / 选人）
  └─ components/.../ArchiveDataImportDialog.tsx  # 通用四步导入弹窗

后端
  ├─ ArchiveDataAdminController           # 统一 REST，勿按模块拆 Controller
  ├─ ArchiveDataAdminService              # 按 path 分发 Handler
  ├─ ArchiveDataResourceRegistry          # path / section / supported
  └─ archivedata/
       ├─ ArchiveDataResourceHandler      # 契约
       ├─ ArchiveDataSupport              # 员工解析、字典、Excel 辅助
       ├─ ArchiveDataModels               # 分页 / 导入结果 DTO
       └─ IdDocumentArchiveDataHandler    # ★ 参考实现（复制套路）
```

**铁律**

1. 写操作（create/update/delete）委托 `EmployeeArchiveService`，与档案 Sheet 共用校验。
2. 新模块只新增 **一个** `*ArchiveDataHandler` + 注册表/前端 `supported=true`，不新建业务表、不新建 Controller。
3. 导出写 **字典名称**；导入兼容 **名称或编码**（见 `ArchiveDataSupport.dictDisplayName` / resolve）。
4. 敏感字段脱敏展示；导出需独立权限 + 审计（沿用现有 Controller 逻辑）。

## 2. 接入清单（每个 resource）

按顺序勾选；一次只接 **1 个** path。

### 2.1 后端

| # | 动作 | 说明 |
| --- | --- | --- |
| 1 | 新建 `{Xxx}ArchiveDataHandler` | `@Component`，`implements ArchiveDataResourceHandler`，`path()` 返回契约 path |
| 2 | 照抄证件 Handler 结构 | `page` → `create`/`update`/`delete` → `buildImportTemplate`/`importExcel`/`exportExcel` |
| 3 | CRUD 调 `EmployeeArchiveService` | 与档案页方法一一对应；禁止直接绕过校验写 Mapper |
| 4 | 定义业务键 upsert | 导入时「已存在则更新」规则写清（如证件：工号 + 证件类型） |
| 5 | Registry `supported=true` | `ArchiveDataResourceRegistry` 对应 path |
| 6 | `mvn compile` | Handler 会被 Spring 注入 `ArchiveDataAdminService` |

**参考类路径**：`server/.../employee/archivedata/IdDocumentArchiveDataHandler.java`

### 2.2 前端

| # | 动作 | 说明 |
| --- | --- | --- |
| 1 | `archive-data-resources.ts` | 设 `supported: true`；补 `listColumns`、`formFields`（复用 `archive-field-defs`） |
| 2 | `importConfig` | `businessKeyHint`、`fillHints`、可选 `fillSubHint` / `templateSheetHint` |
| 3 | Sheet / 列表展示 | 当前页仍以证件为具体表单；复杂字段可先扩页内分支，或后续抽 `ArchiveDataFormFields` |
| 4 | 字典 / 选项 | 需要筛选项时在页内 `getEmployeeFormOptions` 等复用，勿 mock |

**通用能力已有，勿重复实现**

- 导入 UI → `ArchiveDataImportDialog`
- API → `client/src/api/archive-data.ts`
- 权限 → `archiveSectionPermission(section, view|create|edit|delete|import|export)`

### 2.3 验收（最小）

- [ ] 未登录 / 无分区权限 → 403 或无权限卡
- [ ] 列表 loading / error / empty
- [ ] 新建选人（`SearchableDialogPicker`）+ Sheet 保存走档案校验
- [ ] 导入：模板列对齐、业务键 upsert、错误报告可下
- [ ] 导出：字典为名称；审计落库（若现网已接）
- [ ] 档案单人 Sheet 与批管写同一数据，规则一致

## 3. Handler 实现要点（对照证件）

```java
@Component
public class IdDocumentArchiveDataHandler implements ArchiveDataResourceHandler {
  public static final String PATH = "id-documents";

  @Override public String path() { return PATH; }

  // page: 用 support.resolveEmployeeIds 过滤 → Mapper 分页 → toRow（含工号/姓名/部门）
  // create/update: support.resolveEmployeeFromBody → archiveService.createXxx / updateXxx
  // delete: archiveService.deleteXxx
  // 模板/导出: POI 表头与档案字段一致；导出 dictDisplayName
  // 导入: 逐行 try/catch → RowError；字典名/码兼容；是/否 parseYesNo
}
```

**共享工具**（`ArchiveDataSupport`）优先使用：

- `resolveEmployeeIds` / `resolveEmployeeFromBody` / `requireByEmployeeNoForImport`
- `loadEmployees` / `loadOrgNames`
- `dictDisplayName`、日期与是/否解析、Excel 单元格读写

行级导入失败抛 `ArchiveDataModels.RowImportException(field, message)`，由 Handler 汇总为 `ImportResult`。

## 4. 前端资源配置示例（证件）

```ts
{
  path: "id-documents",
  title: "证件信息",
  section: "personal",
  supported: true,
  formFields: PERSONAL_ID_DOCUMENT_FIELDS,
  listColumns: [ /* 工号、姓名、部门、业务列… */ ],
  importConfig: {
    businessKeyHint: "同工号 + 证件类型已存在则更新，否则新建。",
    fillSubHint: "国家/地区、证件类型请填字典名称（与页面展示一致）",
    templateSheetHint: "含「证件信息」数据表与「说明」工作表",
    fillHints: [
      { text: "必填：工号、证件号码" },
      { text: "业务键：同工号 + 证件类型 → 更新；否则新建" },
      { text: "主证件：填写「是 / 否」" },
    ],
  },
}
```

前后端 `supported` **必须同时为 true**，否则：前端显示建设中 / 后端 `requireSupported` 拒绝。
